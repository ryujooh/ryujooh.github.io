# 📇 데이터 분석 포트폴리오 카드 자동 생성 프롬프트

> **이 파일의 용도**
> 데이터 분석 프로젝트를 마칠 때마다, 이 프롬프트를 활용해서
> 깃허브 블로그(`pages/` + `posts.json` 구조)에 포트폴리오 카드를
> 일관된 형식으로 자동 생성하기 위함입니다.
>
> Antigravity의 `docs/` 폴더에 이 파일을 두고, `.agents/` 또는
> Project Rules로 관리하면 AI가 매번 동일한 품질의 결과물을 만들어줍니다.

## 🎯 포트폴리오 카드의 목표

채용 담당자가 **URL 하나를 클릭했을 때, 3분 안에**:

1. 무슨 문제를 풀었는지
2. 어떻게 풀었는지
3. 뭘 발견했는지
4. 어떤 임팩트가 있는지

이걸 다 파악할 수 있도록 만드는 것.

---

## 🔧 AI가 수행해야 할 작업 (3단계)

새 포트폴리오 게시 요청을 받으면 AI는 반드시 **이 3가지를 모두** 수행:

### ✅ Step 1. 마크다운 게시물 생성

- 위치: `pages/{프로젝트-슬러그}.md`
- 슬러그 규칙: 영문 소문자 + 하이픈 (예: `titanic-analysis`, `seoul-apt-trend`)
- 한글 파일명 금지 (URL 깨짐)

### ✅ Step 2. posts.json에 항목 추가

- 기존 `posts.json`을 읽어서
- 새 게시물 항목을 **배열 맨 앞에** 추가 (최신순)
- JSON 형식 깨지지 않게 주의

### ✅ Step 3. 이미지 경로 일관성 유지

- 모든 이미지는 `images/` 루트에 저장
- 마크다운 본문에서 참조: `../images/파일명.png` (pages/에서 보면 상위로 올라가야 하므로)
- 이미지 파일명: `{프로젝트-슬러그}-{용도}.png` (예: `titanic-thumbnail.png`, `titanic-chart1.png`)

## 📋 posts.json 항목 형식

`posts.json`에 추가할 항목 구조 (배열 맨 앞에 삽입):

```json
{
  "slug": "titanic-analysis",
  "title": "타이타닉 생존자 분석 - 누가 살아남았나",
  "summary": "891명 승객 데이터로 생존 요인을 파헤치다",
  "thumbnail": "images/titanic-thumbnail.png",
  "date": "2026-05-12",
  "tags": ["pandas", "EDA", "시각화"],
  "category": "데이터분석",
  "techStack": ["Python", "pandas", "matplotlib", "seaborn"]
}
```

### 필드 설명

| 필드        | 설명                        | 예시                           |
| ----------- | --------------------------- | ------------------------------ |
| `slug`      | URL용 식별자 (영문, 하이픈) | `titanic-analysis`             |
| `title`     | 게시물 제목                 | `타이타닉 생존자 분석`         |
| `summary`   | 한 줄 요약 (목록에 노출)    | `891명 데이터로...`            |
| `thumbnail` | 썸네일 경로 (루트 기준)     | `images/titanic-thumbnail.png` |
| `date`      | 게시일 (YYYY-MM-DD)         | `2026-05-12`                   |
| `tags`      | 검색/필터용 태그 배열       | `["pandas", "EDA"]`            |
| `category`  | 분류                        | `데이터분석`                   |
| `techStack` | 사용 기술 배열              | `["Python", "pandas"]`         |

> ⚠️ **JSON 작성 시 주의**:
>
> - 마지막 항목 뒤에 쉼표 X
> - 한글 문자열은 그대로 입력 (이스케이프 불필요)
> - 기존 항목들 보존, 새 항목만 **맨 앞에** 추가

```
- 프로젝트 제목: 타이타닉 생존자 분석 - 누가 살아남았나
- slug: titanic-analysis
- 사용한 데이터: titanic_train.csv (891명, Kaggle)
- 분석 목적: 어떤 특성이 생존에 영향을 미쳤는지 파악
- 사용 기술: Python, pandas, matplotlib, seaborn
- 주요 발견:
  1. 여성 생존율 74%, 남성 18%
  2. 1등석 생존율 63%, 3등석 24%
  3. 어린이(10세 이하) 생존율 60%
- 차트 1: 성별 생존율 막대 그래프
- 차트 2: 등급별 생존율 막대 그래프
```

- [ ] `pages/{slug}.md` 파일이 생성되었는가?
- [ ] `posts.json`에 항목이 추가되었는가? (JSON 문법 OK)
- [ ] 썸네일 이미지가 `images/{slug}-thumbnail.png`로 저장되었는가?
- [ ] 본문 내 이미지 경로가 `../images/...`로 되어 있는가?
- [ ] slug가 영문 소문자 + 하이픈인가?
- [ ] 한 줄 요약만 읽어도 뭘 했는지 알 수 있는가?
- [ ] 핵심 인사이트 3줄에 **숫자**가 들어가 있는가?
- [ ] 코랩/GitHub 링크가 다 살아있는가?
- [ ] 분석 1, 2, 3 각각에 시각화가 있는가?
- [ ] 결론에 **액션 아이템**이 있는가?
- [ ] Lesson & Learned가 진솔하고 구체적인가?
- [ ] 로컬에서 `post.html?slug={slug}`로 접속했을 때 잘 보이는가?

## 🚨 자주 발생하는 문제 & 해결

### 문제 1. `posts.json`이 깨졌어요

**원인**: 마지막 항목 뒤에 쉼표가 있거나, 중괄호/대괄호가 안 맞음  
**해결**: JSON validator 사이트(jsonlint.com)에 붙여넣어 확인

### 문제 2. 이미지가 안 보여요

**원인**: 경로 오류  
**해결**:

- 본문(pages/\*.md)에서는 `../images/파일명`
- 썸네일(posts.json)에서는 `images/파일명`
- 파일명 대소문자 정확히 일치 확인

### 문제 3. 한글 slug 썼더니 URL이 이상해요

**해결**: 반드시 영문 소문자 + 하이픈만 사용

- `타이타닉` ❌ → `titanic-analysis` ✅
